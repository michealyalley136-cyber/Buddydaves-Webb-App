Buddy Dave's — All URLs (local development)
============================================
Base website:  http://localhost:3000
Base API:      http://localhost:4000

Run locally:     npm run dev              (from project root — starts web + API)
Clean restart:   npm run dev:clean        (fixes blank pages / 500 errors)
Build web only:  npm run build:web        (stop dev first — do not build while dev is running)
Full build:      npm run build            (server + web — stop dev first)

Troubleshooting:
  - Blank screen or 500 (e.g. /menu): run npm run dev:clean, open a NEW tab, Ctrl+Shift+R
  - Browser uses http://localhost:3000 from the terminal (not an old bookmark)
  - Do not run npm run build while npm run dev is running (corrupts web/.next)

================================================================================
API — EXPRESS (port 4000)
================================================================================

Health
  http://localhost:4000/api/health

Database health
  http://localhost:4000/api/health/db

Auth — staff/admin login (POST)
  http://localhost:4000/api/auth/login

Public menu (GET)
  http://localhost:4000/api/menu

Create customer order (POST)
  http://localhost:4000/api/orders

Track order by code (GET)
  http://localhost:4000/api/orders/{code}
  Example: http://localhost:4000/api/orders/BD-20260516-1234

Staff — list orders (GET, requires login token)
  http://localhost:4000/api/staff/orders
  http://localhost:4000/api/staff/orders?view=completed

Staff — update order status (PATCH, requires login token)
  http://localhost:4000/api/staff/orders/{orderId}/status

Staff — demo incoming order for kitchen trial (POST, requires login token)
  http://localhost:4000/api/staff/demo-order

Admin — categories (GET, admin token)
  http://localhost:4000/api/admin/categories

Admin — menu items (GET / POST, admin token)
  http://localhost:4000/api/admin/menu-items
  http://localhost:4000/api/admin/menu-items/{id}     (PATCH / DELETE)

Admin — recent orders (GET, admin token)
  http://localhost:4000/api/admin/recent-orders

Admin — order summary counts (GET, admin token)
  http://localhost:4000/api/admin/orders/summary


================================================================================
API — via website proxy (browser uses these from the Next app)
================================================================================
When using the website in the browser, the app calls same-origin URLs like:
  http://localhost:3000/api/menu
  http://localhost:3000/api/orders
  http://localhost:3000/api/auth/login
  etc.
(Next.js forwards /api/* to http://localhost:4000)
